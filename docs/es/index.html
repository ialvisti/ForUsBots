Descripción general

Este servicio automatiza acciones en portales (actualmente: carga de documentos en el portal de empleadores de ForUsAll) usando Playwright. El servidor inicia sesión (TOTP cuando aplique), completa el formulario, envía el archivo y puede guardar capturas como evidencia. La API recibe el binario y metadatos mínimos (x-meta) y responde 202 Accepted con un jobId.

Docs actualizadas: 2025-08-20

URLs base

  • DEV http://localhost:10000
  • Render https://forusbots.onrender.com

Todas las rutas de la API están bajo el namespace /forusbot.

Autenticación

Las rutas protegidas requieren el header x-auth-token igual a la variable SHARED_TOKEN del servidor. Si falta o no coincide → 401. 

x-auth-token: dev-secret

Por defecto /forusbot/status es público; puedes exigir token cambiando flags.statusPublic con PATCH /forusbot/settings.

Vault File Upload Bot

Automatiza cargas seguras de PDF al portal de empleadores de ForUsAll. Acepta cuerpo binario y metadatos en headers; devuelve 202 Accepted con jobId para consultar.

Endpoint

POST /forusbot/vault-file-upload (auth)

Headers

Nombre Tipo Req Descripción
x-auth-token string sí Token compartido de API.
x-filename string sí Nombre original (p. ej., document.pdf).
x-meta JSON string sí Incluye planId y formData (section, caption, status, effectiveDate).
Content-Type enum sí application/pdf o application/octet-stream.

Body

Binario (el archivo PDF).

Respuestas

  • 202 Accepted — devuelve { ok, jobId, estimate, ... }
  • 400/401/422 — errores de validación/auth

cURL rápido

curl -X POST "$BASE/forusbot/vault-file-upload" \
  -H "x-auth-token: <TOKEN>" \
  -H "x-filename: document.pdf" \
  -H 'x-meta: {"planId":580,"formData":{"section":"CONTRACTS & AGREEMENTS","caption":"Recordkeeper Agreement","status":"Audit Ready","effectiveDate":"2025-05-02"}}' \
  -H "Content-Type: application/pdf" \
  --data-binary @document.pdf

Scrape Participant Bot

Raspa datos del participante por participantId. Soporta selección de módulos y modos de retorno. Devuelve 202 Accepted con jobId; consulta /forusbot/jobs/:id para resultados.

Endpoint

POST /forusbot/scrape-participant (auth)

Body JSON

Campo Tipo Req Descripción
participantId string sí Participante objetivo.
return enum no data|html|text|both (por defecto data).
strict boolean no Validaciones más estrictas.
includeScreens boolean no Adjunta capturas cuando aplique.
timeoutMs number no Timeout global por ejecución.
modules array no Lista de módulos (p. ej., [{"name":"profile"}]).

cURL rápido

curl -X POST "$BASE/forusbot/scrape-participant" \
  -H "x-auth-token: <TOKEN>" -H "Content-Type: application/json" \
  -d '{"participantId":"12345","return":"data","strict":true,"modules":[{"name":"profile"}]}'

Search Participants Bot

Busca participantes con criterios flexibles. Soporta controles de paginación y límites de resultado; devuelve filas normalizadas y un resumen de paginación.

Endpoint

POST /forusbot/search-participants (auth)

Body JSON

Ruta Tipo Req Descripción
criteria.companyName string no Filtra por empresa.
criteria.fullName string no Filtra por nombre completo.
criteria.email string no Email.
criteria.ssn string no SSN (últimos 4 soportados).
criteria.phone string no Teléfono (dígitos).
criteria.participantId string no ID exacto.
options.fetchAllPages boolean no Si es true, recorre páginas hasta pageLimit o maxRows.
options.pageLimit number no Límite duro de páginas (por defecto 1).
options.maxRows number no Máximo de filas a devolver (por defecto 25).
options.timeoutMs number no Timeout general.

Nota: Si maxRows es pequeño, la ejecución puede detenerse tras la primera página aun con fetchAllPages=true.

cURL rápido

curl -X POST "$BASE/forusbot/search-participants" \
  -H "x-auth-token: <TOKEN>" -H "Content-Type: application/json" \
  -d '{"criteria":{"fullName":"miguel"},"options":{"fetchAllPages":true,"pageLimit":5,"maxRows":250}}'

Jobs API — estados y campos

  • Estados: queued, running, succeeded, failed, canceled.
  • Campos: jobId, botId, meta, acceptedAt, startedAt, finishedAt, result, error.
  • Métricas de ejecución: en cola → waitingSeconds y position; ejecutando → elapsedSeconds y etapa; terminado → totalSeconds.

Esquemas

x-meta (compartido)

{
  "planId": number,
  "formData": {
    "section": string,
    "caption": string,
    "captionOtherText"?: string,
    "status": string,
    "effectiveDate": "YYYY-MM-DD"
  },
  "options"?: {
    "returnEvidenceBase64"?: boolean,
    "saveEvidenceToTmp"?: boolean,
    "confirmDelayMs"?: number,
    "fastExit"?: boolean,
    "snifferWindowMs"?: number,
    "suppressSameUrlWarning"?: boolean,
    "evidenceOnSuccess"?: boolean,
    "skipPrevalidation"?: boolean
  }
}

202 Accepted (submit)

{
  "ok": true,
  "jobId": "uuid",
  "acceptedAt": "....",
  "queuePosition": 3,
  "estimate": { "method":"lanes+movingAvg", ... },
  "capacitySnapshot": { ... }
}

Job (GET /jobs/:id)

{
  "ok": true,
  "jobId": "...",
  "state": "running|...|failed",
  "error": null | "mensaje",
  "result": null | {...},
  "waitingSeconds"?: 4,
  "position"?: 1,
  "elapsedSeconds"?: 8,
  "stage"?: "login|otp|...|done",
  "stageSeconds"?: 2,
  "totalSeconds"?: 117
}

Ejemplos cURL

Crear job (202)

META='{"planId":580,"formData":{"section":"CONTRACTS & AGREEMENTS","caption":"Recordkeeper Agreement","status":"Audit Ready","effectiveDate":"2025-05-02"}}'

curl -sS -X POST https://TU_URL/forusbot/vault-file-upload \
  -H 'x-auth-token: TU_TOKEN' \
  -H 'x-filename: presentation.pdf' \
  -H 'Content-Type: application/octet-stream' \
  -H "x-meta: $META" \
  --data-binary @presentation.pdf

Consultar un job

curl -sS -H 'x-auth-token: TU_TOKEN' \
  https://TU_URL/forusbot/jobs/JOB_ID

Listar jobs

curl -sS -H 'x-auth-token: TU_TOKEN' \
  'https://TU_URL/forusbot/jobs?state=queued&limit=20'

Cancelar un job en cola

curl -sS -X DELETE -H 'x-auth-token: TU_TOKEN' \
  https://TU_URL/forusbot/jobs/JOB_ID

Variables de entorno

# Requeridas
PORT=10000
SHARED_TOKEN=dev-secret

# Credenciales del portal
SITE_USER=...
SITE_PASS=...
TOTP_SECRET=...        # Base32, sin espacios
TOTP_STEP_SECONDS=30

# Concurrencia & ETA
MAX_CONCURRENCY=3
ESTIMATE_AVG_SECONDS=120
ESTIMATE_AVG_WINDOW=10

# Evidencias
EVIDENCE_DIR=/tmp/evidence

# Playwright
HEADFUL=0             # 0=headless (recomendado en la nube), 1=mostrar navegador
SLOWMO=0              # ms por acción cuando HEADFUL=1
CLOSE_KILL_MS=1500

El código usa HEADFUL (no HEADLESS). En producción usa HEADFUL=0.

Evidencias

Las capturas se guardan en /tmp/evidence. En dev se exponen vía GET /evidence/*. En producción puedes deshabilitar el acceso público.

Errores & Warnings

  • 400 Bad Request — x-meta inválido o falta el archivo.
  • 401 Unauthorized — token inválido/ausente.
  • 409 Conflict — intento de cancelar un job que ya está corriendo.
  • 422 Unprocessable Entity — regla de negocio rechazada.
  • 500 — error inesperado.
