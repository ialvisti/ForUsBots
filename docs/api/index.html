openapi.yaml v2.2.0 OAS 3.0

Minimal REST endpoints that back the Playwright automation. Submit a file for processing, track job status, adjust concurrency at runtime, and inspect locks & metrics.
All routes are namespaced under /forusbot. A plain /health also exists.

health & status

Simple liveness probe. Also available at /health.

Parameters

No parameters.

Responses

200 OK
{"ok": true}
POST /forusbot/sandbox/vault-file-upload Validate headers/body without launching a job

Dry-run validator. Does not require auth. Validates x-filename and parses/validates x-meta. If a binary body is attached, it is counted but not processed. Returns 200 with a normalized echo or 4xx on validation errors.

Parameters (Headers)

Name In Type Req Description
x-filename header string yes Must end with .pdf.
x-meta header string (JSON) yes Same shape as upload x-meta.
Content-Type header enum no application/octet-stream or application/pdf if binary body is sent.

Responses

200 OK
{
  "ok": true,
  "mode": "dry-run",
  "receivedBinaryBytes": 12345,
  "normalized": {
    "filename": "document.pdf",
    "meta": { "planId": 580, "formData": {"section":"CONTRACTS & AGREEMENTS", "caption":"Recordkeeper Agreement", "status":"Audit Ready", "effectiveDate":"2025-05-02"} }
  },
  "warnings": []
}
400 Bad Request
{
  "ok": false,
  "errorType": "validation",
  "error": "Missing fields in x-meta",
  "missing": ["planId","formData.section","formData.caption","formData.status","formData.effectiveDate"],
  "warnings": []
}
422 Unprocessable Entity 
{
  "ok": false,
  "errorType": "validation",
  "error": "Status 'Document Missing' is not valid when a file is attached"
}

participants

POST /forusbot/scrape-participant Scrape participant modules (queued)

Auth required (x-auth-token). Enqueues a scraping job for a participant. Use GET /forusbot/jobs/:id to poll until completion.

Request Body

{
  "participantId": "12345",
  "return": "data|html|text|both",
  "strict": true,
  "includeScreens": false,
  "timeoutMs": 30000,
  "modules": [
    "census",
    { "key": "payroll", "fields": ["years:all", "2024", "2025"] }
  ]
}

Responses

202 Accepted
{
  "ok": true,
  "jobId": "uuid",
  "acceptedAt": "date-time",
  "queuePosition": 1,
  "estimate": {"method":"lanes+movingAvg","avgDurationSeconds":120,...},
  "capacitySnapshot": {"maxConcurrency":3,"running":0,"queued":0,"slotsAvailable":3},
  "warnings": [
    { "type": "invalid_modules_ignored", "keys": ["unknown"] }
  ],
  "executedBy": { "name": "...", "role": "user|admin", "at": "date-time" }
}

Result shape depends on selected modules. Poll /forusbot/jobs/:id until state: "succeeded".

POST /forusbot/search-participants Search participants (queued)

Auth required (x-auth-token). Normalizes criteria and options, logs in, performs the portal search, and enqueues the job.

Request Body

{
  "criteria": {
    "companyName": "Acme Inc.",
    "fullName": "Jane Doe",
    "email": "jane@acme.com",
    "ssn": "1234",                 // last 4 (or 9)
    "phone": "4155551234",         // 10 digits
    "participantId": "98765"
  },
  "options": {
    "fetchAllPages": true,
    "pageLimit": 3,
    "maxRows": 60,
    "timeoutMs": 12000,
    "evidenceOnSuccess": false
  }
}

Interplay: when fetchAllPages=true, the bot walks subsequent pages up to pageLimit. maxRows caps the total rows returned. The immediate response is 202 Accepted with a jobId; the final job result contains the rows and pagination info below.

Accepted (202)

202 Accepted
{
  "ok": true,
  "jobId": "uuid",
  "acceptedAt": "date-time",
  "queuePosition": 1,
  "estimate": {"method":"lanes+movingAvg","avgDurationSeconds":120,...},
  "capacitySnapshot": {"maxConcurrency":3,"running":0,"queued":0,"slotsAvailable":3}
}

Job result (when succeeded)

{
  "ok": true,
  "targetUrl": "https://employer.forusall.com/view_participants",
  "criteriaEcho": {"companyName": "Acme Inc.", "fullName": "Jane Doe", "email": "jane@acme.com", "ssn": "1234", "phone": "4155551234", "participantId": "98765"},
  "pagination": {
    "pagesFetched": 2,
    "pageLimit": 3,
    "estimatedTotal": 125,
    "hasNextPage": true,
    "shownText": "Showing 1 to 25 of 125 entries"
  },
  "count": 50,
  "rows": [
    {
      "planName": "Acme 401(k)",
      "planId": "580",
      "firstName": "Jane",
      "lastName": "Doe",
      "participantId": "98765",
      "last4Ssn": "1234",
      "status": "Eligible",
      "rmContact": "RM Name",
      "opsContact": "Ops Name"
    }
  ],
  "evidencePath": "/tmp/evidence/search-participants_after_....png"
}
POST /forusbot/mfa-reset Reset MFA for a participant (queued)

Auth required (x-auth-token). Triggers a participant MFA reset and waits for the status to become not enrolled. Returns 202 and you can track progression via the Jobs API.

Request Body

{ "participantId": "12345" }

Responses

202 Accepted
{
  "ok": true,
  "jobId": "uuid",
  "acceptedAt": "date-time",
  "queuePosition": 1,
  "estimate": {"method":"lanes+movingAvg","avgDurationSeconds":120,...},
  "capacitySnapshot": {"maxConcurrency":3,"running":0,"queued":0,"slotsAvailable":3},
  "botId": "forusall-mfa-reset",
  "meta": { "participantId": "12345", "participantUrl": "https://employer.forusall.com/participants/12345" }
}

Poll /forusbot/jobs/:id until the job finishes.

submit

POST /forusbot/vault-file-upload Submit file (binary) → 202 Accepted

Accepts a raw binary body (PDF/Octet-Stream) with headers carrying metadata. The server logs in (TOTP), fills the form, and processes the job asynchronously. Returns 202 with a jobId and ETA.

Parameters (Headers)

Name In Type Req Description
x-auth-token header string yes Must match server env SHARED_TOKEN.
x-filename header string yes Original filename (used for temp write).
x-meta header string (JSON) yes See schema below.
Content-Type header enum yes application/octet-stream or application/pdf.

Request: x-meta (simplified)

{
  "planId": 580,
  "formData": {
    "section": "CONTRACTS & AGREEMENTS",
    "caption": "Recordkeeper Agreement",
    "captionOtherText": "(required only if caption = 'Other')",
    "status": "Audit Ready",
    "effectiveDate": "2025-05-02"
  },
  "options": {
    "confirmDelayMs": 5000,
    "fastExit": false,
    "snifferWindowMs": 7000,
    "evidenceOnSuccess": false
  }
}

Responses

202 Accepted
{
  "ok": true,
  "jobId": "2ef1db9f-9a0d-4b9b-97a1-6b6a0f2e0a90",
  "acceptedAt": "2025-08-20T00:00:00.000Z",
  "queuePosition": 2,
  "estimate": {
    "method": "lanes+movingAvg",
    "avgDurationSeconds": 120,
    "startSeconds": 60,
    "finishSeconds": 180,
    "startAt": "2025-08-20T00:01:00.000Z",
    "finishAt": "2025-08-20T00:03:00.000Z"
  },
  "capacitySnapshot": { "maxConcurrency": 3, "running": 1, "queued": 1, "slotsAvailable": 2 }
}
400 Bad Request
{
  "ok": false,
  "error": "Missing or empty fields in x-meta"
}

{"ok": false, "error": "unauthorized"}

GET /forusbot/jobs List jobs (auth)

Supports filters: state (queued|running|succeeded|failed|canceled), botId, limit, offset.

Parameters (Query)

Name In Type Req Description
state query string no Filter by state.
botId query string no Filter by bot id (e.g. forusall).
limit query integer no Max items (default 50, max 500).
offset query integer no Pagination offset.

Responses

DELETE /forusbot/jobs/:id Cancel queued job (auth)

Only jobs in the queue can be canceled. Running jobs return 409.

Responses

200 OK
{"ok": true, "canceled": true, "reason": "queued"}
409 Conflict
{"ok": false, "error": "No se puede cancelar: job en ejecución"}
404 Not Found
{"ok": false, "error": "Job no encontrado"}
